## [span_2](start_span)Compréhension des Principes SOLID[span_2](end_span)

Voici mon résumé de la compréhension des cinq principes SOLID :

1. **SRP (Single Responsibility Principle) :** Chaque classe doit avoir une seule et unique responsabilité. [span_3](start_span)[span_4](start_span)Cela facilite la maintenance et les tests.[span_3](end_span)[span_4](end_span)
2. **[span_5](start_span)[span_6](start_span)OCP (Open/Closed Principle) :** Une entité logicielle doit être ouverte à l'extension (on peut ajouter des fonctionnalités) mais fermée à la modification (on ne change pas le code existant).[span_5](end_span)[span_6](end_span)
3. **[span_7](start_span)[span_8](start_span)LSP (Liskov Substitution Principle) :** Une classe enfant doit pouvoir remplacer sa classe parente sans casser le fonctionnement du programme.[span_7](end_span)[span_8](end_span)
4. **[span_9](start_span)[span_10](start_span)ISP (Interface Segregation Principle) :** Il vaut mieux avoir plusieurs petites interfaces spécifiques qu'une seule interface générale que les classes sont forcées d'implémenter inutilement.[span_9](end_span)[span_10](end_span)
5. **[span_11](start_span)[span_12](start_span)DIP (Dependency Inversion Principle) :** Il faut dépendre des abstractions (interfaces) et non des implémentations concrètes pour réduire le couplage entre les modules.[span_11](end_span)[span_12](end_span)
